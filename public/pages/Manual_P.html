[t00]Position Adjustment in MSTS Route Editor [t03]Part 1 - Route Editor: [t07]Part 2 - Text Editor: [t14]Part 3 - Route Editor: [t17]Hints and Comments: [t18]Why aren't those path elements combined with track in one object as a track section? [t01]This manual is intended to explain how path elements can take-over the position and direction of a track or road section. Provided that the pivot point and the direction of the static object fits in with the track section, it would be best to have a selection of static objects with equivalent dimensions as common track sections. [t02]The following example shows the construction of an embankment, available from the archive NR_Bahntrasse_1. The package provides various types of path elements, exactly matching the dimensions of A1t*-A4t* tracks available in XTracks 3.20. The file names of the static objects can be easily related to the corresponding track sections. [t04]Place an embankment object with a file name indicating identical dimensions for each track section somewhere nearby. Make sure that you drop it in the same tile as the track section (check the borders by pressing F7). [t05]Mark all embankment objects as Detail level 1 and define it as Terrain object. These settings are available in the Properties dialog window (F6) of a selected item. All tracks sections that have been designated for getting an embankment, should be marked as Detail level 1, too. [t06]Save the route now (it may be sufficient to save world placement changes). Remember the tile identification (tile x) and (tile z) out of the Camera window. [t08]Open the tile's world file in a text editor (e.g. Notetab Light). It is located in the route's world folder, the file name contains the tile identification numbers and the time stamp should be most up-to-date. [t09]Search for the first entry containing the string StaticDetailLevel ( 1 ) (all other entries on this level can be found subsequently) [t10]Mark the two rows Position and Qdirection in the TrackObj description and copy it to the clipboard. [t11]Mark the similar rows in the description of a Static element where FileName ( NR_Emb...) fits the specifications of the track section and paste the content of the clipboard. [t12]Delete the row StaticDetailLevel ( 1 ) in each processed description. This will help you to identify the remaining entries. [t13]Save the *.w file in text editor. [t15]To see the changes in Route Editor it is not necessary to reload the route.
Simply do a broad jump - e.g. decrease one of the tile's numbers by 1000 and click on jump. Then increase the number to the original value and click on jump again. Now you will find the world file reloaded. [t16]Now the embankment fits together with the track. [t19]Objects defined as track sections cannot cast dynamic shadows which is a property of static objects only. Therefore it is the best choice to use static objects as described here, at least for bridges. [t20]Track sections cannot be defined as terrain objects. This may cause an erratic contour where the object reaches into ground. However, this fact may be disregarded for all embankments. So watch out for the second version of this package NR_Bahntrasse_2 which will include tracks. The usage of this objects will be slightly different, so it has its own manual. [t22]Just like track sections seem to be mirrored by pressing 'T', the pivot of the path element will be moved to the other end. If you decide to place such a curved object manually, you have to work in the opposite direction. [t24]If you have questions, comments or suggestions, please contact the author  
[t00]Ajustement de position sous l'éditeur de routes MSTS [t03]Part 1 - éditeur de route: [t07]Part 2 - éditeur de texte: [t14]Part 3 - éditeur de route: [t17]Conseils et commentaires: [t18]Pourquoi ces éléments de chemin ne sont-ils pas combinés avec la voie en un seul objet défini comme section de voie? [t21]Pourquoi y a t-il des courbes orientées vers la droite? [t23]Ou puis-je trouver une aide supplémentaire? [t01]Ce manuel décrit comment des éléments de chemin peuvent permettre de gérer la position et la direction d'une section de voie ou d'une route. Sous réserve que le point pivot et la direction de l'objet statique correspondent à ceux de la section de voie, il est préférable d'avoir une sélection d'objets statiques avec des dimensions équivalentes à celles des sections de voie commune. [t02]L'exemple suivant illustre la pose d'un talus, disponible dans l'archive NR_Bahntrasse_1. Ce pack contient différents types d'éléments de chemin, qui correspondent exactement aux dimensions des voies A1t*-A4t* disponibles dans XTracks 3.20. Les noms de fichier des objets statiques peuvent être aisément mis en correspondance avec ceux des sections de voies. [t04]Pour chaque objet de voie à modifier, placez un objet de talus portant un nom de fichier indiquant les mêmes dimensions que l'objet d'origine. Assurez vous que vous l'insérez bien dans la même tuile (Tiles) que la section de voie d'origine (vérifiez les bords en pressant F7). [t05]Marquez tous les objets de talus comme appartenant au niveau de détail 1 (Detail level 1) et comme étant des objets de terrain (Terrain object). Ces paramètres sont disponibles dans le panneau de dialogue Propriétés (Properties) de chaque objet sélectionné, que l'on affiche avec la touche F6. Toutes les sections de voies désignées pour recevoir un talus doivent également étre marquées comme appartenant au niveau de détail 1 (Detail level 1). [t06]Sauvegardez la route (en fait, il est suffisant de sauvegarder les modifications de placement du monde (world placement changes). Notez l'identifiant de la tuile (Tile X) et (Tile Z) visibles dans la fenêtre Caméra. [t08]Ouvrir le ficher world de la parcelle avec un éditeur de texte (par exemple Notepad++ ou Notetab Light). Ce fichier se trouve dans le répertoire World de la route. Son nom comporte le numéro d'identification de la parcelle, et la date doit étre à jour. [t09]Recherchez la première occurrence du mot clef StaticDetailLevel ( 1 ) (les occurrences suivantes peuvent étre recherchées par la suite). [t10]Marquer les lignes Position et Qdirection dans la description du TrackObj, et copier les dans le presse-papier. [t11]Marquer les mêmes lignes dans la description d'un objet statique dont le suffixe du nom ( NR_Emb... ) correspond aux spécifications de la section de voie, puis coller y le contenu du presse-papier. [t12]Supprimer la ligne StaticDetailLevel ( 1 ) dans chaque description ainsi traitée. Cela vous aidera à repérer les entrées suivantes. [t13]Sauvegarder le fichier *.w depuis l'éditeur de texte. [t15]Afin de vérifier le résultat des manipulations précédentes sous l'éditeur de mode, il n'est pas nécessaire de recharger la route. Il suffit:
  1. d'effectuer un saut important - par exemple, en réduisant de 1000 le numéro de parcelle, et en cliquant Sauter (Jump).
  2. de revenir ensuite à la parcelle initiale en restaurant son identifiant en ajoutant 1000, puis en cliquant à nouveau sur Sauter (Jump).
L'environnement complet est alors rechargé. [t16]Le talus est désormais adapté à la voie. [t19]Les objets définis comme sections de voies ne supportent pas les ombres dynamiques, qui ne s'appliquent qu'à aux objets statiques. Il est donc plus judicieux d'utiliser de tels objets statiques, du moins pour ce qui concerne les ponts. [t20]Les sections de voie ne peuvent pas étre définies comme objets de terrain. Cela peut engendrer des contours erratiques à l'endroit où l'objet rencontre le terrain. Cependant, cela n'est pas vrai pour les talus. Il est donc possible d'utiliser le contenu de la seconde version de ce pack NR_Bahntrasse_2, qui contient ces voies. L'usage de ces objets est légèrement différent, et fait l'objet d'un manuel spécifique. [t22]Comme les sections de voie apparaissent en miroir avec la touche "T", le point pivot se trouve déporté à l'autre extrémité.
