[t00]Gleisobjekt-Austausch im MSTS-Streckeneditor [t01]In dieser Anleitung soll erklärt werden, wie Gleisobjekte durch andere (Gleis-)Objekte ausgetauscht werden können. Dies können Gleisobjekte mit modifizierter Texturzuweisung sein oder aber auch Gleiselemente mit direkt integrierten Kunstbauten wie Bahndämmen oder Stützmauern. [t02]Das nachfolgende Beispiel zeigt den Einbau eines Bahndamms aus dem Paket NR_Bahntrasse_2. Das Paket enthält verschiedene Arten von Kunstbauten für Bahntrassen (Bahndämme, Rampen und Stützmauern) inklusive der Gleiskörper für ein- bis viergleisige Strecke, die in Abmessung und Namensgebung leicht den jeweiligen ("A"-)Gleistücken aus XTracks 3.20 zugeordnet werden können. [t03]Teil 1 - im Streckeneditor: [t04]Die Strecke, die einen Böschungs­körper erhalten soll, wird ganz normal aufgebaut. [t05]Alle Gleisstücke, die einen Böschungskörper erhalten sollen, werden angeklickt bzw. ausgewählt und über das Eigenschaftsfenster (F6) der Detailstufe 1 zuge­ordnet. [t06]Nun wird die Strecke gespeichert. Es genügt, Änderungen an der Außenwelt­platzierung zu speichern.
Es ist hilfreich, sich die Planquadrat-Kennzahlen aus dem Kamera-fenster zu merken (Planq. x und Planq. z)
[t07]Teil 2 - im Texteditor: [t08]In einem Text­editor (z.B. Notetab Light) wird die .W-datei des be­arbei­te­ten Planquadrats geöffnet (sie befindet sich im WORLD-Ordner der Strecke).
Der Dateiname enthält die Planquadrat-Kennzahlen, die Datei hat ein sehr aktuelles Änderungsdatum.
[t09]Nun wird der erste Eintrag mit StaticDetailLevel ( 1 ) gesucht: (alle Einträge eines Detail Levels stehen untereinander.) [t10]In der Beschreibung des Gleis­stücks (TrackObj) soll nun der Ver­weis auf die verbundene Shape-Datei geändert werden.
Dazu wird der für alle weiteren Objekte gleich bleibende Teil des FileName-Eintrags markiert.
Anschließend wird der "Suchen + Ersetzen"-Dialog aufgerufen (Menü: Search - Replace... oder einfach: Strg+R) [t11]Im Dialogfeld Replace with wird das Suchmuster (blau) mit dem relati­ven Pfad zum Austauschobjekt er­gänzt (orange).
Dabei muss be­dacht werden, dass ein TrackObj immer im Ordner Global/Shapes erwartet wird.
Um zur Order­struktur der bear­bei­teten Strecke zu gelangen, muss man also zu­nächst zwei Ebenen zurück (..\\..\\), um dann in den Routes-Ordner, die be­ar­beitete Strecke und den darin enthaltenen 'Shapes'-Ordner zu verweisen.
Der Ordnername der bearbeiteten Strecke (hellgrün) muss natürlich dem richtigen Namen angepasst werden.
Achtung!
Der Name des Streckenordners darf keine Leerzeichen enthalten. Sonst muss der in Klammern stehende Eintrag nach FileName ( ".. von Anführungszeichen umschlossen werden" ). Ich em­pfehle, bei Datei- oder Ordnerna­men generell auf Leerzeichen zu verzichten! [t12]Anschließend werden die Zeilen mit StaticDetailLevel ( 1 ) gelöscht, damit ersichtlich ist, dass dieser Eintrag nicht mehr barbeitet werden muss. [t13]Nun kann die .W-Datei im Texteditor gespeichert werden.

[t14]Teil 3 - im Streckeneditor: [t15]Um das Ergebnis im Streckeneditor betrachten zu können, muss die Strecke nicht neu geladen werden.
Es reicht aus, im Kamerafenster einen weiten Sprung zu unternehmen - z.B. um 1000 Quadrate. Anschließend springt man wieder zurück - und die .W-Datei wurde neu geladen. [t16]Nun sind die Standardgleise er­setzt durch Gleise mit Bahndamm.
In diesem Zustand lassen sie sich im Streckeneditor behandeln wie jedes andere Gleisstück auch. D.h. sie können anderswo platziert, geneigt oder gespiegelt werden.
[t17]Anmerkungen und Hinweise: [t18]Warum gibt es in diesem Paket keine Einträge für die REF-Datei der Strecke ("addREF.txt")? [t19]Austauschobjekte sind nicht dafür vorgesehen, als statische Objekte im Streckeneditor platziert zu werden. Sie werden ausschließlich durch Umadressierung in den World-Dateien in die Strecke eingefügt. [t20]Warum sollten Brücken nicht auf diese Art und Weise als Gleisobjekte integriert werden? [t21]Gleisobjekte können keine dynamischen Schatten anzeigen. Das ist nur statischen Objekten möglich. Daher ist diese Bauart z.B. für Brücken die bessere Variante. [t22]Wo kann ich weitere Unterstützung erhalten? [t23]Für weitergehende Fragen stehe ich gern im TSSF-Forum zur Verfügung (Anmeldung erforderlich).
[t00]Track Shape Substitution in MSTS Route Editor [t01]This manual is intended to explain how track sections can be replaced with other shape objects (including track). This can be a modification of an original track shape with another texture assignment or a track shape combined with a path element like an embankment or retaining wall. [t02]The following example shows the the substitution of standard track with a track piece attached on an embankment, available from the archive NR_Bahntrasse_2. The package provides track sections combined with various types of path elements (embankments, ramps and retaining walls), exactly matching the dimensions of A1t*-A4t* tracks available in XTracks 3.20. [t03]Part 1 - Route Editor: [t04]Place an embankment object with a file name indicating identical dimensions for each track section somewhere nearby. Make sure that you drop it in the same tile as the track section (check the borders by pressing F7). [t05]Mark all embankment objects as Detail level 1 and define it as Terrain object. These settings are available in the Properties dialog window (F6) of a selected item. All tracks sections that have been designated for getting an embankment, should be marked as Detail level 1, too. [t06]Save the route now (it may be sufficient to save world placement changes). Remember the tile identification (tile x) and (tile z) out of the Camera window. [t07]Part 2 - Text Editor: [t08]Open the tile's world file in a text editor (e.g. Notetab Light). It is located in the route's world folder, the file name contains the tile identification numbers and the time stamp should be most up-to-date. [t09]Search for the first entry containing the string StaticDetailLevel ( 1 ) (all other entries on this level can be found subsequently) [t10]The next step is to change the FileName entry in the TrackObj description.
At first, mark the FileName row from the start to the last char that remains unchanged for all subsequent entries.
Open the "Find and replace" dialog window (Menu: Search - Replace... or shortcut key: Ctrl+R) [t11]Copy the search string from the Find what to the Replace with field (blue). Insert the relative path to the substitution file (orange). Be sure to utilize double backslashs.
Remember that the standard location of a TrackObj is always the folder Global/Shapes.
To get to your route in process, you have to step up two levels first (..\\..\\), enter the Routes folder, your route and the Shapes subfolder.
The name of the route's folder (light green) has to match your route's folder name.
Please note:
The name of your route's folder should not contain whitespaces. Otherwise the whole path has to be put in "quotation marks". My hint: Never use whitespaces in file names or folder names. [t12]Delete the row StaticDetailLevel (1) in each processed description. This will help you to identify the remain­ing entries. [t13]Save the *.w file in text editor. [t14]Part 3 - Route Editor: [t15]To see the changes in Route Editor it is not necessary to reload the route.
Simply do a broad jump - e.g. decrease one of the tile's numbers by 1000 and click on jump. Then increase the number to the original value and click on jump again. Now you will find the world file reloaded. [t16]Now the standard tracks are replaced by embankment tracks.
These objects can be moved, sloped or aligned just like standard track.
[t17]Hints and Comments: [t18]Why are there no REF file entries included in this archive? [t19]These objects are intended for substitution of existing track objects, not for placing as static objects in Route Editor. [t20]Why are there no bridge objects combined with track as a track section? [t21]Objects defined as track sections cannot cast dynamic shadows which is a property of static objects only. Therefore it is the best choice to use static objects as described here, at least for bridges. [t22]Where can I get more assistance? [t23]If you have questions, comments or suggestions, please send me an email  
[t00]Substitution des formes de voies dans les routes MSTS [t01]Ce manuel décrit comment les sections de voie peuvent être remplacées par d'autres objets de forme (y compris des objets de voie). A une voie, on pourra ainsi en substituer une autre portant une texture différente, ou encore combinée à élément de décor, tel un talus ou un mur de souténement. [t02]L'exemple suivant illustre le remplacement d'éléments de voie standard par des éléments similaires associés avec un talus. Cet élément est disponible dans l'archive NR_Bahntrasse_2. Ce pack propose des sections de voie combinées avec divers type de plateformes (talus, rampes, murs de souténement), correspondant exactement aux dimensions des voies A1t*-A4t* disponibles dans XTracks 3.20. [t03]Part 1 - Éditeur de route: [t04]Pour chaque objet de voie à modifier, placez un objet de talus portant un nom de fichier indiquant les mêmes dimensions que l'objet d'origine. Assurez vous que vous l'insérez bien dans la même tuile (Tiles) que la section de voie d'origine (vérifiez les bords en pressant F7). [t05]Marquez tous les objets de talus comme appartenant au niveau de détail 1 (Detail level 1) et comme étant des objets de terrain (Terrain object). Ces paramètres sont disponibles dans le panneau de dialogue Propriétés (Properties) de chaque objet sélectionné, que l'on affiche avec la touche F6. Toutes les sections de voies désignées pour recevoir un talus doivent également être marquées comme appartenant au niveau de détail 1 (Detail level 1). [t06]Sauvegardez la route (en fait, il est suffisant de sauvegarder les modifications de placement du monde (world placement changes). Notez l'identifiant de la tuile (Tile X) et (Tile Z) visibles dans la fenêtre Caméra. [t07]Part 2 - Éditeur de texte: [t08]Ouvrir le ficher world de la parcelle avec un éditeur de texte (par exemple Notepad++ ou Notetab Light). Ce fichier se trouve dans le répertoire World de la route. Son nom comporte le numéro d'identification de la parcelle, et la date doit être à jour. [t09]Recherchez la première occurrence du mot clef StaticDetailLevel ( 1 ) (les occurrences suivantes peuvent être recherchées par la suite). [t10]L'étape suivante consiste à modifier le nom du fichier (FileName) dans la section TrackObj. En premier lieu, sélectionnez la ligne FileName depuis le début jusqu'au dernier caractère qui doit restez inchangé pour toutes les occurrences suivantes. Ensuite, ouvrez le menu Chercher et remplacer de l'éditeur de texte, afin de rechercher le texte sélectionné à l'étape précédente dans le fichier. [t11]Recopier le contenu du champ Rechercher (Find What) vers le champ Remplacer avec (Replace with). Sur la figure ci-contre, ce contenu est surligné en bleu. Insérer le chemin relatif vers le fichier, surligné en orange. Assurez vous de bien utiliser les doubles backslashs entre chacun des niveaux du chemin. Souvenez vous aussi que l'emplacement standard d'un objet de voie (TrackObj) est toujours le répertoire Global/Shapes. Ainsi, le chemin vers la route à traiter doit contenir:
  1. tout d'abord, la remontée de deux niveaux (..\\..\\),
  2. ensuite le répertoire de la route,
  3. enfin le sous-répertoire Shapes de la route.
Bien entendu, le nom du répertoire My_Routes (surligné en vert) doit être adapté en fonction du nom de votre route!
Remarque importante:
Si le nom du répertoire de votre route contient des espaces, alors le chemin enter doit être encadré par des "doubles guillemets". Cela dit, idéalement, les noms de répertoires ou de fichiers ne devraient pas contenir d'espaces! [t12]Supprimer la ligne StaticDetailLevel (1) dans chaque description à traiter. Cela aidera à identifier les entrées restant à traiter. [t13]Sauvegarder le fichier *.w depuis l'éditeur de texte utilisé. [t14]Part 3 - Éditeur de route: [t15]Afin de vérifier le résultat des manipulations précédentes sous l'éditeur de mode, il n'est pas nécessaire de recharger la route. Il suffit:
  1. d'effectuer un saut important - par exemple, en réduisant de 1000 le numéro de parcelle, et en cliquant Sauter (Jump).
  2. de revenir ensuite à la parcelle initiale en restaurant son identifiant en ajoutant 1000, puis en cliquant à nouveau sur Sauter (Jump).
L'environnement complet est alors rechargé. [t16]La voie standard a bien été remplacée par la voie sur talus.
Ces objets peuvent être déplacés, inclinés, ou alignés exactement comme une voie normale.
[t17]Conseils et commentaires: [t18]Pourquoi il n'ya pas les entrées du fichier REF inclus dans cette archive? [t19]Ces objets ont vocation à remplacer des voies existantes, et non pas à être placés comme objets statiques dans l'éditeur de route!. [t20]Pourquoi il n'ya pas d'objets de pont combiné avec piste comme une section de voie? [t21]Les objets définis comme sections de voies ne supportent pas les ombres dynamiques, qui ne s'appliquent qu'aux objets statiques. Il est donc plus judicieux d'utiliser de tels objets statiques, du moins pour ce qui concerne les ponts. [t22]Où puis-je obtenir plus d'aide?
