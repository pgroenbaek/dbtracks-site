[t04]Erste Schritte mit Dynatrax   ( zum Lesen bitte klicken ) [t05]

 
Konvertierung von "Dynamischen Schienen" (Dyntrack) in statische Gleisobjekte:
  • Einstellungen bei Dynatrax: Wenn keine MSTS Installations im ersten Auswahlfeld angezeigt werden, muss ein MSTS-Programmverzeichnis über den 'Add'-Knopf gesucht und hinzugefügt werden.
  • Die Strecke, die bearbeitet werden soll, wird im 'Route'-Auswahlfeld eingestellt. Das Programm beginnt danach sofort, die World-Dateien nach Dyntracks zu durchsuchen. Am unteren Fensterrand sieht man dazu eine Fortschrittsanzeige.
  • Die Dynatrax-Profildatei, die zur Anwendung kommen soll, wird unter 'Package' ausgewählt.
    Aus diesem Paket muss nun ein 'Track Profile' ausgewählt werden.
  • Einen oder mehrere (Strg+Klick) 'Shape'-Einträge in der 'Dynamic Track Status'-Liste auswählen und 'Convert' drücken.
[t06]Arbeiten mit verschiedenen Profilen   ( zum Lesen bitte klicken ) [t07]
  • Wenn es mehr als ein 'Dyntrack' in der World-Datei gibt und nur eines davon konvertiert werden soll, muss im Streckeneditor die Position des Gleisstücks - und anschließend in der World-Datei die UiD ermittelt werden.
    Tipp: In der Gleisplan-Ansicht von "Dynatrax v1.0 beta" sieht man die UiD in der Statusleiste, wenn man ein Gleis mit dem Mauszeiger berührt.
    Um die Dateizuordnungen normaler Gleisstücke auszutauschen, ist die "Alias"-Funktion von "Dynatrax v1.0 beta" eine große Hilfe.
  • Wenn ein bereits konvertiertes Gleisstück mit einem anderen 'Track Profile' umgewandelt werden soll und man dessen UiD ermittelt hat, markiert man den betreffenden Eintrag, wählt ein anderes 'Track Profile' und drückt den Knopf 'Rebuild'.
  • Wenn man die "dynamische Schiene" wieder herstellen will (z.B. um die Eigenschaften zu ändern), markiert man den betreffenden Eintrag und drückt den Knopf 'Restore'.
" [t08]Arbeiten mit 'DBTracks' Profilen [t09]
 
Profil-Sets wie 'NR DB Track Basic' enthalten eine Menge sogenannter 'Track profiles'. Das sind Formvorlagen, die die verschiedenen Varianten der DBTracks, bzw. verschiedene Qualitätsstufen davon erzeugen können. Hierbei gilt es ein paar Dinge zu bachten:
  • 'Track profile'-Einträge beginnend mit '(q200)' oder '(q150)' erhöhen die Strukturqualität des Gleisobjekts auf die angegebene Prozentzahl. Das kann bei engen Kurven notwendig sein.
  • 'Track profile'-Einträge beginnend mit '(q75)' oder '(q50)' verringern die Strukturqualität des Gleisobjekts auf die angegebene Prozentzahl. Damit reduziert sich die Anzahl der Polygone, so dass auch sehr große Objekte im MSTS geladen werden können.
  • 'Track profile'-Einträge beginnend mit '(s...)' können den Schwellenabstand verlängern oder verkürzen, um ein Gleisstück nicht inmitten einer Schwelle enden zu lassen.

Hinweise zur Anwendung:

Wenn die Dateigröße eines umgewandelte Dyntrack 500K übersteigt (siehe Abbildung), kann es sein, dass das Gleisstück im MSTS nicht geladen werden kann. Dann empfiehlt sich ein 'Rebuild' mit dem '(q75)' Profil. In extremeren Fällen kann das '(q50)' Profil helfen, um Objekte mit noch passender Polygonzahl zu erzeugen. Diese verringerte Qualität ist aber bei Kurven deutlich erkennbar.

Um die Integrität dieser großen Dateien zu prüfen, empfiehlt es sich, im (gleichzeitig geöffneten) Streckeneditor in die entsprechenden Planquadrate zu springen. Nicht ladbare Gleisstücke würden dann sofort gemeldet.

[t04]First Steps With Dynatrax   ( please click to open ) [t05]

 
Convert dynamic track (dyntrack) to a static track shape:
  • Settings in Dynatrax: If there are no MSTS Installations shown in the first select box, search and add your MSTS folder by pressing the 'Add' button.
  • Choose the route you want to edit in the 'Route' select box. The program will start locating the contained Dyntracks automatically. A progress bar is shown at the bottom of the window.
  • Choose the Dynatrax profile package file that you want to use in the 'Package' select box.
    Now you can select the 'Track Profile' you want to use next in the box below.
  • Pick and mark 'Shape' entries (add more with ctrl+click) in the 'Dynamic Track Status' list and press the 'Convert' button.
[t06]Working With Different Profiles   ( please click to open ) [t07]
  • If there is more than one dynamic track in your world file and you want to convert only one of them, search for the UiD in the world file and check the position in Route Editor.
    Handy hint: The track plan view of "Dynatrax v1.0 beta" will show the UiD in the status bar, if you point with the mouse on a track section.
    To replace file names of normal track sections, the "Alias" function of "Dynatrax v1.0 beta" is a great facilitation.
  • If you want to alter the track profile and you have found the UiD, mark the regarding entry, choose another track profile and press 'Rebuild'.
  • If you want to have the dynamic track back (e.g. to edit the properties), mark the regarding entry and press the 'Restore' button.
[t08]Working With 'DBTracks' Profiles [t09]
 
Profile packages like 'NR DB Track Basic' contain a lot of different 'Track profiles'. These are blueprints not only for diffent types of DBTacks but also for different qualities. There are some things to remark:
  • 'Track profile' names starting with '(q200)' or '(q150)' increase the mesh structure quality by the specified percentage. This may be necessary for sharp curves.
  • 'Track profile' names starting with '(q75)' or '(q50)' decrease the mesh structure quality by the specified percentage. This results in less polygons in order to match the MSTS restrictions.
  • 'Track profile' names starting with '(s...)' can adjust the distance of sleepers in order (to try) to avoid a track shape with a truncated sleeper.

A hint for quick reference:

If a converted track has a file size greater than 500K it is possible that the track section can not be loaded. You may rebuild it with the '(q75)' profile. Even larger files can be converted with the '(q50)' profile to create objects with acceptable polygon count. However the reduced quality is clearly evident, especially on bends.

To check the integrity of this large files it is recommended to use the route editor. Jumping to a tile number will indicate an oversized object immediately.
